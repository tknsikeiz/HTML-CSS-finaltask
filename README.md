# PetLife - ペット情報・飼い方ガイド

## 概要
PetLife は、ペットとの暮らしを支える情報サイトです。テンプレート7ページを修正し、
ログイン・新規登録ページを追加して計9ページの構成にしました。

## ページ構成（9ページ）
- ホーム: index.html
- ペット図鑑: pet-guide-list.html
- ペット詳細: pet-guide-detail.html
- 飼い方ガイド: pet-care-guide.html
- Q&A: pet-guide-qa.html
- 飼育記録: pet-guide-record.html
- 記録投稿: pet-guide-form.html
- ログイン（新規）: login.html
- 新規登録（新規）: signup.html

## 使用したBootstrapコンポーネント
以下のコンポーネントを10種類以上使用しています。

- Navbar（全ページ）
- Grid（Container / Row / Col）
- Cards（一覧・特集表示）
- Accordion（飼い方ガイド / ペット詳細）
- Carousel（ペット詳細の画像切替）
- Tabs / Pills（Q&Aカテゴリ）

- Forms（検索・登録・投稿フォーム）
- Progress Bars（評価や強度表示）
- Badges（カテゴリ・状態表示）
- Buttons / Button group

【バグ1】ナビゲーションバーのトグルアイコン非表示
────────────────────────────────────────────────
  問題: スマートフォンサイズで、ハンバーガーメニューアイコンが見えない
  
  原因: 
    Bootstrapの`.navbar-toggler-icon`は、背景画像で表示される仕組みです。
    `.navbar-light`または`.navbar-dark`クラスがないと、背景色が指定されないため、
    アイコンが表示されません。
  
  修正方法:
    <nav class="navbar navbar-expand-lg sticky-top">
    
    ↓↓↓
    
    <nav class="navbar navbar-expand-lg navbar-light sticky-top">
  
  影響ファイル: 7ページすべて
  
  使用技術: Bootstrap 5 Navbarコンポーネント


【バグ2】ナビゲーションリンクが機能しない（#でダミー化）
────────────────────────────────────────────────
  問題: クリックしてもページ遷移しない
  
  原因: 
    すべてのナビゲーションリンクが `href="#"` に設定されていた
  
  修正内容:
    ホーム              → index.html
    ペット図鑑          → pet-guide-list.html
    飼い方ガイド        → pet-care-guide.html
    Q&A                → pet-guide-qa.html
    飼育記録            → pet-guide-record.html
  
  さらに、ページごとに `active` クラスを付与し、
  現在位置がわかるようにしました。
  
  使用技術: HTML基本（アンカータグの href 属性）


【バグ3】パンくずリストがダミーの不具合（詳細ページ）
────────────────────────────────────────────────
  ファイル: pet-guide-detail.html
  
  修正:
    ホーム       → index.html
    ペット図鑑   → pet-guide-list.html
    犬           → pet-guide-list.html（カテゴリページは未用意のため）

【改善1】ホームページの統計情報セクション

    </div>

【改善2】ログイン・新規登録フォームのレイアウト
────────────────────────────────────────────────
  
  ファイル: login.html, signup.html
  
  改善前:
    - フォーム要素が全幅で表示される
    - 登録ページが長すぎて見づらい
  
  改善後:
    
    ■ ログインフォーム (login.html)
      - 背景: グラデーション
      - Card を中央配置（max-width: 400px）
      - アイコン + タイトル + フォーム
      - ソーシャルログインボタン
      - レスポンシブ: モバイルで自動スタック
    
    ■ 登録フォーム (signup.html)
      - CSS Grid でお名前を2列配置
        grid-template-columns: 1fr 1fr
        Desktop: 並列 / Mobile: 積層
      
      - ペット選択をグリッドで表示
        grid-template-columns: repeat(auto-fit, minmax(100px, 1fr))
        → 画面幅に応じて自動折り返し
        → 6個のペットオプションが1行～複数行に配置
      
      - パスワード強度インジケーター
        長さに応じて色が変わる
        弱（赤）→ 中（黄）→ 強（緑）
  
  コード例:
    <!-- お名前を2列配置 -->
    <div class="row">
      <div class="col-md-6">
        <input class="form-control" placeholder="姓">
      </div>
      <div class="col-md-6">
        <input class="form-control" placeholder="名">
      </div>
    </div>
    
    <!-- ペット選択を自動折り返し -->
    <div style="display: grid; grid-template-columns: repeat(auto-fit, minmax(100px, 1fr)); gap: 10px;">
      <label class="form-check">
        <input class="form-check-input" type="checkbox" value="dog">
        <i class="bi bi-paw-fill"></i> 犬
      </label>
      <!-- ... -->
    </div>
  
  効果:
    - フォーム入力が効率的
    - 画面サイズに自動適応
    - モダンなデザイン
    - 直感的なユーザーインターフェース

  - 重複を削減してコード保守性を向上させるため
  - ファイルサイズを削減してページロード速度を向上
  - 1箇所の変更が全ページに反映されるため、拡張性が向上
  - 実務ではスタンダードな方法

Q: "レイアウト改善の2箇所はどこか？"
A:
  - ① ホームページの統計情報を Bootstrap Grid の4列配置で表現
  - ② ログイン・登録フォームで CSS Grid を使用した複数要素の配置

Q: "ファビコンはなぜ SVG にしたのか？"
A:
  - PNG より軽い（ファイルサイズ小）
  - 拡大してもぼやけない（ベクトル形式）
  - レティナディスプレイ対応

Q: "どのBootstrapコンポーネントを使用したか？"
A:
  Navbar / Grid / Card / Accordion / Carousel / Tabs / Breadcrumb / 
  Toast / Forms / Progress
